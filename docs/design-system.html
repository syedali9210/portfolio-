Design system

The tokens, type, motion and voice behind syedali.dev — documented from the actual source, not redrawn from memory. Currently feeling: Design engineer.

/Foundations

Two themes, one system. Dark is the default — the real ThemeProvider ships defaultTheme="dark" — and light mode is a considered second pass, not an inverted afterthought. Everything below is a real CSS custom property from globals.css; toggle the switch in the top bar to see both.

Neutrals
Surfaces — an 8-step elevation ramp
Context: this ramp does real work in dark mode (#171717 → #484848, a visible climb used to stack panels without borders) but goes almost flat in light mode (five of eight steps are plain #FFFFFF). That's not an oversight — light mode leans on card/border tokens instead, because dark is where this site actually lives.
Accent — the pet buddy's palette
The only saturated color in the whole system, and it's fixed across both themes — it belongs to the mascot, not the UI chrome. It shows up exactly three places: the pet buddy's body, the hero tagline text, and a one-frame terracotta glow on the theme-toggle transition below.
Elevation
Two different illusions, on purpose. Light mode fakes elevation the flat-design way — the same soft rgba(0,0,0,.06) shadow, stacked and re-stacked at growing blur radii. Dark mode fakes it the physical way instead: a hairline inset ring for the edge, a faint top highlight for "light catching the bevel," and only then a real drop shadow to ground it. Same 8-step scale, different material.
Radius
One knob — --radius: 0.625rem — scaled by a fixed multiplier for every step (sm ×0.6 up to 4xl ×2.6). Change the one value and the whole site's corners move together.
/Type

Body copy runs on Segoe UI — self-hosted, all five weights, subsetted from a 5,400-glyph system font dump down to the ~230 characters the site actually renders. It's a deliberately un-fashionable choice: no Inter, no Geist Sans, the literal Windows UI face, worn like the "I do things the unconventional way" line in the hero means it.

Scale
Tailwind's default scale, with exactly one override: the base step drops from 16px to 14px site-wide (line-height 1.6). text-xl — 20px — is reserved for exactly two things: the name in the hero card, and every section's /Heading.

/Motion

One easing curve does almost everything on the site — scroll, fades, the accordion — so motion always feels like it belongs to the same hand. Two exceptions get their own physics, on purpose: a bouncier spring for a photo popping into a spotlight, and a synthesized click for the one control that flips the whole world.

The house ease — cubic-bezier(.22, 1, .36, 1)
Shared, byte-for-byte, by Lenis' smooth scroll, every Framer Motion fade, and the yapping accordion. A fast start that glides to a soft stop — never a bounce, never linear.
Scroll reveal — FadeIn
opacity 0→1, y 16px→0, 600ms, house ease, fires once per element the first time it enters view. Siblings stagger in ~50ms steps so a row of cards doesn't arrive as one flat sheet.
Spotlight spring — stiffness 260 / damping 22
The one deliberately playful motion in the system: tap a sticker in About me and it pops into a full-screen spotlight on an underdamped spring — it's allowed to overshoot a little before settling, unlike everything else here.
Theme toggle — a real click, a real wipe
The switch in the top bar is wired exactly like the live site: a synthesized square-wave chirp (2200Hz→900Hz, ~70ms, Web Audio — no audio file), then a circular View Transitions wipe that blooms a terracotta drop-shadow at 35% before settling.
/Voice

First person, dry, a little self-aware. Never corporate. Section headings borrow a /slash-path convention — like breadcrumbs in a terminal — because the site is written by someone who is as much engineer as designer, and wants you to clock that in the first four characters of every heading.

The asterisk aside

A lowercase, asterisk-prefixed caption for anything that needs a one-line stage direction — always quiet, always text-muted-foreground, never a full sentence with a capital letter:

*drag and play *yapp a bit more *Scratch the card to reveal it, then hover to see it in action.
/Layout

One column, 680px wide, centered — on a 27" monitor or a phone, the site reads at the same measure. There is no wide desktop layout to speak of; width past 680px is margin, not content.

px-4 sm:px-6 — 16 / 24px inset
py-12 sm:py-20 — 48 / 80px section rhythm
max-width: 680px
screen-line-top / screen-line-bottom — a hairline-grid utility class applied to nearly every section and card in the codebase, currently rendering nothing (the stroke is switched off by design). It's kept everywhere as a no-op on purpose: the grid can be switched back on globally without touching a single component's spacing. Think non-printing guides left in, just in case.

/Mascot

The pet buddy: the closest thing this brand has to a logo character. Built entirely from axis-aligned rectangles on a shared 9.375px unit grid — no curves anywhere — then given a illusion of depth with two darker, offset copies of its own silhouette underneath, and a uniform outline via an SVG feMorphology dilate filter. It's alive below: it tracks your cursor, reaches an arm out when you stray to one side, blinks every few seconds, and cheers if you click it.

highlight → body → shadow → outline. Four flat fills, zero gradients — the "3D" is an optical trick, not a render.
